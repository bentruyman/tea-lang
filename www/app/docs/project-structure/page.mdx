# Project Structure

Learn how to organize your Tea projects for maintainability and scalability. This guide covers recommended directory layouts and file naming conventions.

## Basic Project Layout

A typical Tea project follows this structure. Tea is flexible about organization, but this layout works well for most projects.

<Card className="p-6 bg-card border-border">

```
my-project/
├── src/
│   ├── main.tea          # Entry point
│   ├── config.tea        # Configuration
│   └── utils/
│       ├── mod.tea       # Module definition
│       └── helpers.tea   # Helper functions
├── tests/
│   └── utils_test.tea    # Test files
├── examples/
│   └── demo.tea          # Example programs
└── README.md             # Documentation
```

</Card>

## Key Directories

<DirectoryCard icon="folder" title="src/" description="Contains your main source code. The entry point is typically main.tea or named after your project.">

```tea
# src/main.tea
use utils = "./utils/mod"

def main() -> Void
  @println(utils.greet("World"))
end

main()
```

</DirectoryCard>

<DirectoryCard icon="file" title="Module Files (mod.tea)" description="Each directory that acts as a module should have a mod.tea file that exports public functions.">

```tea
# src/utils/mod.tea
pub def greet(name: String) -> String
  `Hello, ${name}!`
end

pub def add(a: Int, b: Int) -> Int
  a + b
end
```

</DirectoryCard>

<DirectoryCard icon="test" title="tests/" description="Test files are typically placed in a tests/ directory. Use the std.assert module for assertions.">

```tea
# tests/utils_test.tea
use assert = "std.assert"
use utils = "../src/utils/mod"

assert.eq(utils.add(1, 2), 3)
assert.eq(utils.greet("Tea"), "Hello, Tea!")
```

</DirectoryCard>

## Module Imports

Tea supports both standard library imports and relative path imports.

<Card className="p-6 bg-card border-border">
  <div className="space-y-4">
    <div>
      <h3 className="font-semibold text-accent mb-2">Standard Library</h3>

```tea
use fs = "std.fs"
use json = "std.json"
use path = "std.path"
```

</div>
    <div>
      <h3 className="font-semibold text-accent mb-2">Relative Imports</h3>

```tea
use helpers = "./helpers"
use utils = "../utils/mod"
```

</div>
  </div>
</Card>

## Larger Project Example

For larger projects, you might organize by feature or domain.

<Card className="p-6 bg-card border-border">

```
my-app/
├── src/
│   ├── main.tea
│   ├── config/
│   │   └── mod.tea
│   ├── models/
│   │   ├── mod.tea
│   │   ├── user.tea
│   │   └── team.tea
│   ├── services/
│   │   ├── mod.tea
│   │   ├── auth.tea
│   │   └── data.tea
│   └── utils/
│       ├── mod.tea
│       └── helpers.tea
├── tests/
│   ├── models_test.tea
│   └── services_test.tea
└── examples/
    └── demo.tea
```

</Card>
